# Support

This document explains where to ask questions, how to report problems, and which channel to use for different kinds of requests.

## Get Help

### Usage questions
Use GitHub Discussions if enabled for this repository. Discussions are the best place for:

- setup questions
- model routing questions
- configuration help
- architecture and deployment trade-offs
- brainstorming around improvements

If Discussions are not enabled, open a GitHub issue only when you can describe a concrete bug or actionable documentation gap.

### Bug reports
Use the GitHub issue tracker and select the bug report template.

Include:

- your operating system
- Docker / Docker Compose version
- whether you are using CPU-only or GPU mode
- exact steps to reproduce
- relevant logs and error messages

### Feature requests
Use the GitHub issue tracker and select the feature request template.

Good requests explain:

- the problem to solve
- the proposed behavior
- alternatives already considered
- examples of how users would interact with the feature

## Security

Do **not** open public issues for vulnerabilities.

Follow the private reporting guidance in [`SECURITY.md`](SECURITY.md).

## Response Expectations

This is an open-source project maintained on a best-effort basis.

Typical expectations:

- security reports: initial acknowledgement target within 48 hours
- bug reports and feature requests: triaged as maintainer time allows
- pull requests: reviewed as maintainer time allows

No guaranteed SLA is provided.
